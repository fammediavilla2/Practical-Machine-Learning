Practical Machine Learning Project

Introduction:

This is an R Markdown document subitted to fulfill the requirements to complete the Practical Machine Learning Course.

We are mining the data from: http://groupware.les.inf.puc-rio.br/har

For this exercise I decided not to use the preprocesed data; namely the measures of location (i.e., quantiles, mean, medians) and dispersion provided (i.e., ranges and standard deviations).

In its stead, I used the variables which seem to be raw measurements (i.e, roll, pitch, yaw, gyros, accel, and magnet) for all sensors (i.e, belt, arm, and dumbbell).

That accounts for a total of 38 variables; thus reducing the number of features from 160.

Preliminary Analysis of the Reduced Set

Examining some of these pairs of variables reveals that there is a marked separation among classe, for these features.

All of the features are continuous variables. Categories D and E seem very numerous and seem to form separate clusters, when examined based on the dimansions (roll, pitch and yaw).

Method

Based on the characteristics of the features, scale of measurements and their level of separation, I decided to try either Support Vector Machines or a Multinomial Logistic Regression

to construct the classification model.

Follows my implementation of Logistic Regression found in the RWeka package, as implemented through the cater package.

Twenty five bootstrap samples were used for cross-validation, to select the best model.

Expected Performance on Out of Sample

The performance on the training set was acceptable in predicting all categories of classe.

The in-sample error is estimated to be (0.999, 1) with a 95% confidence interval.

The lowest accurracy for the bootstrap samples (25 of them) was 100%; therefore, I estimated an out of sample

error rate of (1-max(accuracy)) 0%. However, this is not realistic. On other runs I estimated error rates of up to 4%.

I expected the performance on the testing set to be worse, mainly due to overfitting.

As it turned out, when the results were submitted, the model achieved 100% accuracy for the testing set consisting of 20 observations.
